# Switch model from Sonnet 4.6 to Opus 4.7

## Context

The book-reader currently pins `claude-sonnet-4-6` as the only model used by the AI assistant (single-turn Q&A about selected PDF regions). The user wants to try Opus 4.7 — the latest and most capable Claude model — as the default. Motivation is curiosity / wanting to see if the newest model noticeably improves answer quality, not a specific quality complaint or a context-ceiling issue.

Tradeoff acknowledged: Opus is meaningfully more expensive per token than Sonnet, and the workload (single-turn `maxTurns: 1`, mostly short questions about a passage) is well within Sonnet's strengths. Going with the 200k variant (not the 1M variant) so we don't have to reason about the larger context window in the gauge.

## Change

Single-line edit. `MODEL_NAME` is the only source of truth for the model id; everything else (`lib/claude.ts`, `ContextUsageGauge`, the `MODEL_MAX_TOKENS` map) reads through it.

### File: `lib/contextWindows.ts`

```ts
// before
export const MODEL_NAME = "claude-sonnet-4-6";

// after
export const MODEL_NAME = "claude-opus-4-7";
```

The `MODEL_MAX_TOKENS` map is keyed off `[MODEL_NAME]`, so the 200k entry automatically applies to the new id — no map edit needed. (Opus 4.7's standard variant is 200k; the 1M variant uses a different id `claude-opus-4-7[1m]` which we are not adopting here.)

## Files NOT changed (and why)

- `lib/claude.ts:28` — already reads `MODEL_NAME`, no edit.
- `components/ConversationPanel.tsx:1483` — already passes `MODEL_NAME` to the gauge.
- `components/ContextUsageGauge.tsx` — receives `model` as a prop, no hardcoded id.
- `docs/plans/2026-04-28-01-oh-book-reader-plan.md`, `docs/plans/2026-05-02-23-rename-claude-to-ai-in-ui.md`, `docs/plans/2026-05-02-36-context-usage-badge.md` — historical/dated plan documents that are snapshots of past work; updating them would rewrite history. Leave alone.
